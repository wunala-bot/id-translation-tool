# 证件翻译自动化工具 — 产品设计文档

> **目标：** 用户上传证件照片 → 系统自动提取信息并翻译 → 生成标准格式英文文件
> **定位：** 免费工具，打破签证材料翻译的信息差垄断

---

## 一、支持的证件类型（V1.0）

### Phase 1（当前）
| 证件类型 | 页面数 | 核心字段 |
|---------|--------|---------|
| **身份证** | 2页（正面/背面） | 签发机关、有效期、姓名、性别、民族、出生日期、住址、身份证号 |
| **户口本** | 2页（首页/个人页） | 户别、户主、户号、住址、签发日期、姓名、关系、出生地、籍贯、文化程度、婚姻状况等 |

### Phase 2（后续扩展）
| 证件类型 | 复杂度 | 说明 |
|---------|--------|------|
| 护照 | 低 | 个人信息页，字段与身份证类似 |
| 驾照 | 低 | 正副页，字段标准化 |
| 出生医学证明 | 中 | 需特殊字段（出生时间、孕周、接生机构等） |
| 结婚证 | 低 | 持证人信息、登记日期 |
| 房产证 | 中 | 产权人、坐落、面积、用途等 |
| 银行流水 | 高 | 表格形式，需OCR识别表格结构 |
| 在职证明 | 中 | 自由文本，需NLP提取关键信息 |

## 二、用户体验流程（一步到位）

```
┌─────────────────────────────────────────────────────────────┐
│  步骤1：选择证件类型（身份证/户口本/护照...）                │
│         ↓                                                   │
│  步骤2：上传照片（支持拖拽，自动识别正反面）                  │
│         ↓                                                   │
│  步骤3：【OCR自动识别】提取文字信息（预留接口）               │
│         ↓                                                   │
│  步骤4：【自动翻译】填充英文字段（用户可校对修改）            │
│         ↓                                                   │
│  步骤5：预览确认 → 导出PDF/Word                             │
└─────────────────────────────────────────────────────────────┘
```

### 关键体验设计

**1. 最少操作原则**
- 用户只需：选类型 → 上传照片 → 点导出
- OCR识别后的字段自动填充，用户只需校对（不是从头填写）
- 自动翻译实时生效，无需额外点击

**2. 智能默认值**
- 民族、性别、血型等标准选项 → 直接字典匹配
- 日期格式 → 自动转换为英文格式（1990年1月1日 → January 1, 1990）
- 地址 → 保留中文地名拼音 + 替换行政区划后缀（市→City, 区→District）

**3. 可编辑性**
- 所有自动识别/翻译的结果都可手动修改
- 英文框用蓝色背景标识，提示"这是自动翻译的，请校对"
- 姓名拼音必须手动输入（系统提示 "Pinyin required"）

**4. 批量处理**
- 支持同时上传多张证件（如身份证正反面+户口本首页+个人页）
- 系统自动分类到对应页面

## 三、技术架构

### 前端
- **纯静态HTML/JS**（当前版本），可部署到 GitHub Pages / Vercel / Netlify
- **html2pdf.js** 生成PDF（客户端生成，无需后端）
- **响应式设计**：支持手机/平板/电脑

### OCR接口（预留）

| 方案 | 优点 | 缺点 | 推荐度 |
|-----|------|------|--------|
| **百度智能云文字识别** | 中文识别准确率高，有身份证/银行卡专用接口 | 需实名认证，有免费额度限制 | ⭐⭐⭐⭐⭐ |
| **腾讯云OCR** | 支持身份证、户口本专用识别，结构化输出 | 价格略高 | ⭐⭐⭐⭐ |
| **阿里云OCR** | 文档结构化识别强 | 学习成本略高 | ⭐⭐⭐⭐ |
| **Google Vision API** | 多语言支持好 | 国内访问需翻墙 | ⭐⭐⭐ |
| **Tesseract.js** | 纯前端，免费，无需API Key | 中文识别率一般，需训练 | ⭐⭐ |

**推荐方案：百度智能云**
- 身份证识别专用接口，返回结构化JSON（姓名、性别、民族、地址等字段已分离）
- 每月有免费额度（通常1000次/月）
- 识别准确率 > 98%
